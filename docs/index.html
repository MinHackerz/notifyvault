Application Specifications

App Name: NotifyVault

Service Type: Local Notification Logger & Database
Data Server Sync: None (100% Offline-only)

1. Information Collection and Use

NotifyVault uses the Android NotificationListenerService API to read incoming system notifications. The app captures and saves details of incoming notifications (such as Title, Text, Subtext, App Name, Package Name, and timestamp) directly into a local SQLite database on your device.

Strict Privacy Protection: All captured notifications are stored entirely on your local device. We do not run any cloud servers, and we never collect, upload, share, or transmit your database logs to any external systems or third-party servers.

2. Advertisements

To support ongoing development, NotifyVault displays third-party advertisements using the Google Mobile Ads SDK (AdMob).

Google AdMob may collect and process device advertising identifiers (such as the Android Ad ID) and location metrics to serve personalized or non-personalized banner ads. The collection and usage of this data are governed by Google's Privacy Policy.

3. Data Retention and Deletion

By default, the database stores notification history locally. You are in complete control of your data:

  • Retention Settings: You can configure automatic database cleanup (e.g. automatically purge messages older than 7 days) inside the Settings screen.
  • Manual Wipe: You can erase your entire notification history database instantly by tapping the "Clear Database" button in the Settings menu.
  • App Uninstall: Uninstalling the NotifyVault application from your Android device immediately deletes the entire SQLite database file and all stored logs from your disk permanently.

4. Security

Since your database is offline and stored locally, it is protected by Android’s standard sandboxing architecture. No other applications on your device can access NotifyVault's private database directory.
